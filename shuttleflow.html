getSpecificUserOperationList

Get user mint/burn operations list of specific user.

Parameters

(1) data:(JSON Object) the query detail 

{
    "type": "mint" | "burn"

    "address": user conflux address
    "status": ["doing", "finished"]
}

(2) offset:(Number) the number of leading operations to be ignored

(3) count:(Number) the maximum number of operations to return, <= 100

Returns

Explanation:

common fields for all operation status:

status: "confirming" (wait eth/btc/cfx confirmation) | "doing" (custodian signing & settle on chain) | "finished"

user_addr: user conflux address

to_addr: user conflux address (for mint), or user bit/eth address (for burn)

settled_tx: transaction sent by custodian alliance to process user operation. If this field is not empty, it denotes that the user operation is settled on chain and waiting for confirmation.

special status for confirming and doing status:

signer: list of custodian members who already signed

need: total number of signatures

has_signed: Don't use this field to determine operation status! This field denotes whether the custodian member hosting the rpc node has signed the operation or not.

Token Sponsor Backend RPC List

getTokenList

Get shuttleflow supported token list.

Parameters

None.

Returns

List of Objects: 

{
    "reference": (String) "eth" | "btc" | lowercase erc20 address
    "reference_name": (String) erc20 name (none for btc/eth)
    "reference_symbol": (String) erc20 symbol (none for btc/eth)
    "reference_name": (String) erc20 name
    "reference_symbol": (String) erc20 symbol
    "burn_fee": (String) fee set by token sponsor(captain), taken from user when burn cToken
    "mint_fee": (String) fee set by token sponsor(captain), taken from user when mint cToken
    "wallet_fee": (String) fee set by token sponsor(captain), taken from user when mint cToken, for the deployment of receive wallet
    "minimal_mint_value": (String), set by token sponsor, minimal acceptable mint value
    "minimal_burn_value": (String), set by token sponsor, minimal acceptable burn value
    "symbol": (String) symbol of cToken, e.g. "cBTC", "cETH", "cUSDT", can be duplicated
    "decimals": (Number), decimals of reference, e.g. for btc is 8, for eth is 18, for usdt is 6
    "sponsor": (String), conflux address of current token sponsor
    "sponsor_value": (String) amount of mortgaged cETH of current token sponsor
    "ctoken": (String) contract address of cToken
    "is_admin": (bool) is admin token or not, admin token's sponsor can only be custodian alliance whitelist admin, normal user is unable to replace it.
    "icon": (?) token icon
}

burn cToken from sender

function burn(
        address user_addr, // user conflux address
        uint256 amount, // burn amount
        uint256 expected_fee, // expected burn fee, in 18 decimals
        string memory addr, // external chain receive address
        address defi_relayer // external chain defi relayer address
) public;

Burn cToken from sender's address.

user_addr is used to logging, in case Conflux Defi burn cToken for normal user.

amount should be larger than burn_fee and minimal_burn_value in CustodianCore contract(Note here the decimals of amount is 18 and decimals of burn_fee and minimal_burn_value in CustoidanCore contract is reference token's decimal, the smart contract will do the decimal conversion for comparison). Besides, if the fee is higher than expected_fee, the burn transaction will revert.

addr is a ethereum or bitcoin address, which the reference token will be sent to.

If it is a burnt of eth/erc20 tokens, user can specify a defi_relayer, which will send the eth/erc20 to defi_relayer and logging the user's information. See more in defi relayer.

If user does not want to use defi_relayer, fill it with zero address.

For example,burn(cusdt_address, 1e18, 1e16, user_eth_addr, zero_addr) will burn 1 cUSDT and user is willing to pay no more than 0.01 cUSDT as burn fee to sponsor. The burned usdt will directly sent to user_eth_addr.

For example,

cETH address => 18,

cBTC address => 8,

cUSDT address => 6.

reference token => is admin

mapping(string => bool) public admin_token;

Denote each cToken is admin token or not.

Admin token can only be sponsored by whitelist admin of custodian core contract, and will maintain both hot and cold wallet. For non-admin token, anyone can be its sponsor and only hot wallet will be maintained.

reference token => cToken address

mapping(string => address) reference_token; // inverse token_reference

Anti mapping of token_reference.

reference token => burn fee (except btc)

mapping(string => uint256) public burn_fee;

The fee taken from user during cToken burn. Note the decimals is decimals of the reference token.

For example, the sponsor want to take 0.1 cBTC when user burn cBTC, here burn_fee['btc'] should be 0.1 * 1e8.

reference token => mint fee (except btc)

mapping(string => uint256) public mint_fee;

The fee taken from user during cToken mint. Note the decimals is decimals of the reference token.

reference token => wallet fee (except btc)

mapping(string => uint256) public wallet_fee;

When a user want to migrate token from ethereum to conflux, his/her receive wallet will be deployed firstly if it is not deployed before. wallet_fee denote the fee taken from user during deployment of receive wallet. Note the decimals is decimals of the reference token.

reference token => minimal mint value (except btc)

mapping(string => uint256) public minimal_mint_value;

The remaining value of reference token in user's receive wallet must be larger than minimal mint value, otherwise the custodian alliance will not mint for it.

Note: for btc, use btc_minimal_mint_value.

minimal mint value for bitcoin

uint256 public btc_minimal_mint_value

cBTC minimal mint value

reference token => minimal burn value (except btc)

mapping(string => uint256) public minimal_burn_value;

The minimal burn value of corresponding cToken, the burn will fail if burn value is not enough.

Note: for btc, use btc_minimal_burn_value.

minimal burn value for bitcoin

uint256 public btc_minimal_burn_value;

sponsor token (be token captain)

function sponsorToken(
        string memory ref, // lowercase erc20 ethereum address
        uint256 amount, // amount of cETH mortgaged
        uint256 _burn_fee, // burn fee of erc20 token, in erc20 decimals
        uint256 _mint_fee, // mint fee of erc20 token, in erc20 decimals
        uint256 _wallet_fee, // receive wallet fee of erc20 token, in erc20 decimals
        uint256 _minimal_mint_value, // minimal mint value of erc20 token, in erc20 decimals
        uint256 _minimal_burn_value // minimal burn value of erc20 token, in erc20 decimals
) public;

This function can be called by anyone.

This function helps msg.sender become the sponsor of an erc20 token, specified by ref.

The governance contract will transfer amount cETH from msg.sender by operator authority, and compare the amount to the current sponsor of ref(if exists), the sponsor will be replaced by msg.sender if amount is large enough. The token parameters of ref will also be replaced by the parameters sender provided, if the token is not in cooldown.

modify token parameters (only token sponsor)

function setTokenParams(

        uint256 _burn_fee,
        uint256 _mint_fee,
        uint256 _wallet_fee,
        uint256 _minimal_mint_value,
        uint256 _minimal_burn_value
) public;

This function can be called by anyone.
